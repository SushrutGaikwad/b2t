# Per-Frame Conversion with a Deterministic Scaffold - Design

Date: 2026-06-16
Status: proposed

## 1. Motivation

Today the `convert` node hands the entire flattened, overlay-free Beamer deck to
one LLM call and gets back a whole Typst deck, preamble and all. That has three
problems as the roadmap grows:

- The model carries the full deck as context on every call, which dilutes
  attention and makes the upcoming math/units/chem/theorem routing (roadmap item
  2) harder to target.
- The fixed preamble (imports, theme setup, title slide, outline, bibliography,
  thank-you slide) is regenerated by the model each time, even though it is
  deterministic boilerplate keyed on a few facts we already extract.
- There is no natural seam for the per-frame human review the roadmap calls for
  (item 4).

This design restructures the pipeline to convert one Beamer frame at a time and
to build all fixed structure deterministically. The model is left with exactly
one job per call: translate a single frame body. The cyclic graph shape this
introduces is also the exact shape the later HITL increment will interrupt on.

## 2. Scope

In scope:

- Split the stripped source into a preamble, deck metadata, structural flags,
  and an ordered list of frames (each tagged with its section).
- A deterministic scaffold/assembly that builds the Typst header, optional
  outline, optional bibliography, and thank-you slide, and interleaves section
  headings with the converted frames.
- Deterministic title-metadata extraction (`\title`, `\subtitle`, `\author`,
  `\institute`, `\date`) into `config-info`.
- Deterministic `.bib` detection and copy (new; `includes.py` does not parse
  `\addbibresource`/`\bibliography` today).
- Per-frame LLM conversion that runs as a real LangGraph cycle, automatically,
  with no human approval between frames.
- Prompt `convert/v3`, scoped to a single frame.

Out of scope (separate, later increments):

- Per-frame human-in-the-loop approve / regenerate / feedback, the persistent
  checkpointer it needs, and the UI changes to drive it (the immediate next
  spec).
- The compile-error fix loop.
- `\subsection` mapping to its own heading.
- Full bibliography normalization (roadmap item 5); only detection, copy, and
  the bibliography block are done here.
- Visualization packages, UA-1 tagging, and the SaaS wrapper.

## 3. Decisions already made (with rationale)

| Decision | Choice | Why |
| --- | --- | --- |
| Spec scope | Restructure to per-frame plus the deterministic scaffold now; defer the per-frame HITL approval loop to its own spec | Keeps this increment mostly deterministic and fully testable, and keeps the heavy UI/checkpointer work isolated |
| Per-frame loop shape | A real LangGraph cycle (`convert_frame` plus a conditional edge over a frame index), not a single node looping internally | It is the exact structure the HITL increment will interrupt on, so the cyclic shape is built once |
| Heading ownership | The LLM emits `== Frame Title` plus body per frame; the scaffold inserts `= Section` from `\section` | Frame titles can contain LaTeX/math, so the model handles them; section titles are plain text and stay deterministic |
| Title metadata | Deterministic parse of all five fields; `\date` renders to `datetime(...)` when it parses to a real date, else `datetime.today()` with the raw text kept in a comment | Matches the input where it safely can, never emits invalid Typst |
| In-frame citations | The model maps `\cite`/`\citep`/`\citet` to `@key`, guided by the prompt; deterministic code only detects/copies the `.bib` and emits the bibliography block | Inline citation conversion is semantic and local to the frame body the model already owns; full normalization is a later item |
| Full reference per frame | Keep sending the whole reference deck on every frame call | Correctness now; trimming the reference is a later optimization, noted below |

## 4. Architecture overview

```
copy_input -> clean_build -> detect_main -> flatten -> strip_overlays
   -> split_deck
   -> convert --(frame_index < len(frames))--> convert   [LLM, cycle]
        |
   (frames exhausted)
        v
   -> assemble -> write_output -> compile -> END
```

`copy_input`, `clean_build`, `detect_main`, `flatten`, `strip_overlays` are
unchanged. `split_deck` and `assemble` are new deterministic nodes. The LLM node
stays **registered as `convert`** (implemented by a new per-frame function) and
is the only LLM node; it is bound to the `LLMClient` via `functools.partial`
exactly as today. Keeping the registered name `convert` matters: `run_prompt`,
`llm_choices`, `llm_runs`, the prompt directory `prompts/convert/`, and the
`/api/graph` `is_llm` flag are all keyed off the graph node's name, so renaming
the node would silently break the UI's LLM-node detection and the per-node
pickers. Only its behavior changes (one frame per invocation, in a cycle).

## 5. State model (`src/b2t/state.py`)

Two new submodels:

```python
class DeckMeta(BaseModel):
    title: str | None = None
    subtitle: str | None = None
    author: str | None = None
    institute: str | None = None
    date_raw: str | None = None        # the raw \date argument, rendered at assembly

class FrameUnit(BaseModel):
    raw: str                           # the whole \begin{frame}...\end{frame}
    section: str | None = None         # the \section this frame falls under, if any
```

`PipelineState` gains:

```python
    # split_deck
    preamble: str | None = None        # text before \begin{document}; LLM context
    meta: DeckMeta | None = None
    has_toc: bool = False              # a \tableofcontents frame was present
    bib_file: Path | None = None       # detected .bib, if any
    frames: list[FrameUnit] = Field(default_factory=list)

    # convert_frame (the cycle)
    frame_index: int = 0               # cursor into frames
    converted_frames: list[str] = Field(default_factory=list)  # == Title + body, in order
```

`typst_source` keeps its current meaning: the final assembled deck, set by
`assemble`. `aspect_ratio` and `image_files` are unchanged.

`convert_frame` reads `converted_frames` from the incoming state, appends its one
result, and returns the full updated list plus `frame_index + 1`. This keeps the
existing "node returns full dicts, last writer wins per key" convention and needs
no LangGraph reducer.

## 6. Components

### 6.1 `split_deck` (deterministic, `nodes/split_deck.py` + `latex/split.py`)

Input: `stripped_tex`, `work_dir`. Output: `preamble`, `meta`, `has_toc`,
`bib_file`, `frames`.

Pure logic lives in `latex/split.py` (network-free, tested directly); the node is
a thin adapter, matching the existing `latex/` + `nodes/` split.

1. `preamble` = everything before `\begin{document}`.
2. `meta` = regex-parse `\title`, `\subtitle`, `\author`, `\institute`, `\date`
   from the preamble. Missing fields stay `None`. `date_raw` holds the raw
   argument text.
3. Walk the document body in order, tracking the current `\section{...}` /
   `\section*{...}`. For each `\begin{frame}...\end{frame}`, capture its raw text
   tagged with the current section.
   - A frame containing `\titlepage` is excluded (the scaffold renders it with
     `#title-slide()`).
   - A frame containing `\tableofcontents` is excluded and sets `has_toc = True`.
   - Frames before the first `\section` get `section = None` (no `=` heading).
4. `bib_file` = resolve `\addbibresource{X.bib}` or `\bibliography{X}` against
   `work_dir` (new helper in `latex/includes.py`); `None` if absent.

Frames are delimited by matching `\begin{frame}` to its `\end{frame}`; frames do
not nest. The splitter does not parse frame titles (the model owns those). On an
ambiguous or unmatched delimiter it fails loudly rather than guessing, per the
project's "never guess content" rule.

### 6.2 The LLM cycle node (registered as `convert`, `nodes/convert_frame.py`)

A new `convert_frame(state, client)` function, registered in the graph under the
name `convert` (`add_node("convert", partial(convert_frame, client=client))`).
For `frames[frame_index]`, it calls `run_prompt(state, "convert", client,
values)` with `values = {reference, guides, preamble, frame}`. It appends the
model output (after `strip_code_fence`) to `converted_frames`, increments
`frame_index`, and records provenance into `llm_runs`/`llm_rendered` as today.

A router function decides the next edge:

```python
def more_frames(state) -> str:
    return "convert" if state.frame_index < len(state.frames) else "assemble"
```

wired with `add_conditional_edges("convert", more_frames, {"convert": "convert",
"assemble": "assemble"})`. The existing `nodes/convert.py` (whole-deck) is
replaced by this per-frame node.

### 6.3 `convert/v3` prompt

New version; `v1`/`v2` kept for history. `defaults.json` points `convert` at
`v3`. Tokens: `{{reference}}`, `{{guides}}`, `{{preamble}}`, `{{frame}}`.

The system instruction directs the model to:

- output the Typst for this one frame only: a `== Frame Title` heading followed
  by the converted body;
- emit no imports, no theme setup, no `#title-slide()`, no preamble (the scaffold
  owns all fixed structure);
- use the supplied Beamer preamble only as context for custom macros;
- map `\cite{key}`, `\citep{key}`, `\citet{key}` to `@key` inline;
- never use overlays or pause functionality.

The `aspect_ratio` token is dropped from this prompt because the scaffold now
owns the theme call.

### 6.4 `assemble` (deterministic, `nodes/assemble.py` + `typst_scaffold.py`)

Builds `typst_source` from `meta`, `aspect_ratio`, `has_toc`, `converted_frames`,
and `bib_file`. Template logic lives in `typst_scaffold.py` (tested directly).

1. Header: the imports, `show-theorion`, `university-theme.with(aspect-ratio:
   "<aspect_ratio>", config-info(... from meta ...))`, the numbly comment block,
   the text settings, and `#title-slide()`. `config-info` fields come from `meta`;
   absent fields fall back to the reference deck's placeholder text. `date_raw`
   is rendered by a small `render_date` helper that tries a short set of known
   formats (`YYYY-MM-DD`, `Month DD, YYYY`, `Month YYYY`): it emits
   `datetime(year, month, day)`, defaulting the day to `1` when only a month and
   year are given. Anything else (including `\today` or free text) falls back to
   `datetime.today()` with the original text kept in a trailing comment.
2. If `has_toc`: the outline block
   (`= Outline <touying:hidden>` / `== Outline <touying:hidden>` /
   `#components.adaptive-columns(outline(title: none, indent: 1em))`).
3. Body: walk `converted_frames` in order. Track the section of each frame; when
   it differs from the previous frame's section (and is not `None`), emit
   `= <section>` before the frame's Typst.
4. If `bib_file`: append `= References` / `== References <touying:hidden>` /
   `#bibliography("<bib_file.name>", title: none, style: "apa")`, then the
   thank-you slide.

### 6.5 `write_output` / `compile`

`write_output` additionally copies `bib_file` (when set) into `output_dir`, so
`#bibliography("<name>")` resolves. Existing image copy and `image()` path fixing
are unchanged. `compile` is unchanged.

## 7. Data flow (one run)

1. `split_deck` produces `preamble`, `meta`, `has_toc`, `bib_file`, and the
   `frames` list (titlepage and TOC frames excluded), `frame_index = 0`.
2. `convert_frame` runs once per frame: model translates frame `i` to
   `== Title` + body, appends it, advances the index; the conditional edge loops
   until all frames are converted.
3. `assemble` builds the full deck: header, optional outline, section-interleaved
   frame bodies, optional bibliography and thank-you slide.
4. `write_output` writes `main.typ`, copies images and the `.bib`.
5. `compile` runs `typst compile` and records the result.

## 8. Error handling

- No `\begin{document}`, or an unmatched `\begin{frame}`: raise in `split_deck`
  (fail loud, do not guess).
- A deck with zero convertible frames after exclusions: raise, since there is
  nothing to convert.
- A `\bibliography`/`\addbibresource` target that does not resolve to a file:
  raise (a referenced bibliography that is missing is a real error).
- Model/network errors in `convert_frame`: propagate to the existing job failure
  boundary (`failed`), unchanged.
- A `\date` that does not parse: not an error; falls back to `datetime.today()`.
- Compile failure: recorded in `compile_error`, not retried (the fix loop is a
  later item), unchanged.

## 9. Testing

- `tests/test_split.py` (new): preamble extraction; metadata parse including
  missing fields; `has_toc` detection; `.bib` detection and resolution; frame
  list with section tags; titlepage and TOC exclusion; fail-loud on unmatched
  frame delimiters.
- `tests/test_scaffold.py` (new): header with and without TOC; `render_date`
  parse and fallback; section-heading insertion between frames; no-section
  frames; bibliography and thank-you appended only when a `.bib` exists.
- `tests/test_nodes.py`: `split_deck`, `convert_frame` (with `FakeClient`,
  asserting the cycle appends and advances), and `assemble`.
- `tests/test_graph.py`: the per-frame cycle runs N times under `FakeClient` and
  assembles a full deck, on deck1 (no bib) and deck2 (bib). Replaces the
  single-`convert` assertion.
- `tests/test_state.py`: the new fields and submodels.
- `tests/test_prompts.py`: `convert/v3` loads and renders with the new token set.
- Integration (typst present): deck1 and deck2 assemble and compile.

All non-integration tests stay offline via `FakeClient`. The fixtures already
exist: deck1 (plain, no bibliography) and deck2 (biblatex + `references.bib`).

## 10. Known limitations / deferred

- The testing UI renders the pipeline as a fixed linear strip, so the
  `convert_frame` cycle will not display cleanly. Logged here; fixed in the
  HITL/UI increment.
- `\subsection` is not mapped to its own heading; only `\section` to `=`. The
  in-scope fixtures have no subsections.
- Fragile/verbatim frames and the `\frame{...}` shorthand are best-effort; the
  splitter fails loud on ambiguous delimiting rather than guessing.
- The full reference deck is sent on every frame call, so token cost scales with
  frame count. Acceptable for correctness now; trimming the reference for
  per-frame calls is a future optimization.
- Per-frame HITL approve / regenerate / feedback is the separate next spec.

## 11. Future extensions (not built now)

- Per-frame HITL with a persistent checkpointer and the UI to drive it.
- A compile-error fix loop that can re-run a single frame.
- `\subsection` heading support.
- A trimmed, body-only reference for per-frame calls to cut token cost.
